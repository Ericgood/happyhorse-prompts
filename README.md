## 🐴 What is HappyHorse?

**HappyHorse-1.0** is a 15B-parameter unified Transformer that jointly generates video and audio from text — no post-production dubbing required. It emerged in early April 2026 as an anonymous entry on the [Artificial Analysis](https://artificialanalysis.ai/) Video Arena leaderboard, quickly claiming the **#1 spot** and surpassing established models like Seedance 2.0, Kling 3.0, and PixVerse V6.

### Key Highlights

- 🏆 **#1 on AI Video Arena** — Elo 1333 (Text-to-Video), Elo 1392 (Image-to-Video)
- 🧠 **15B Parameters** — 40-layer single-stream Transformer, no cross-attention
- 🎬 **Joint Video + Audio** — Dialogue, ambient sound, and Foley generated alongside video frames
- 🗣️ **Multilingual Lip-Sync** — Chinese, English, Japanese, Korean, German, French, Cantonese
- ⚡ **8 Denoising Steps** — Fast inference via DMD-2 distillation, no CFG required
- 📖 **Open Source** — Base model, distilled model, super-resolution module, and inference code (with commercial-use rights)

### Architecture

The model uses a **single-stream self-attention Transformer** where text tokens, reference image latents, and noisy video/audio tokens are jointly denoised in one sequence. The first and last 4 layers use modality-specific projections; the middle 32 layers share parameters across all modalities.

> *"No one in the open-source community has ever done real audio-video joint pre-training from scratch before."*
>
> — [36Kr report on HappyHorse](https://eu.36kr.com/en/p/3757826958635781)

### The Mystery

Nobody has publicly confirmed who built HappyHorse-1.0. Artificial Analysis described it as a **"pseudonymous" model**. Community investigation suggests it may be an iterative optimization based on the open-source [daVinci-MagiHuman](https://github.com/BrightXiaoHan/MagiHuman) model, potentially linked to Sand.ai and Shanghai's GAIR lab. The naming coincides with 2026 being the **Year of the Horse** in the Chinese zodiac.

## 🤝 How to Contribute

We welcome prompt contributions from everyone! Here's how:

### Submit via Pull Request

1. **Fork** this repository
2. **Add your prompt** — create a Markdown file in the appropriate `prompts/<category>/` folder using the template below
3. **Submit a PR** — we'll review and merge it!

### Prompt Template

Create a file like `prompts/cinematic/my-awesome-prompt.md`:

```markdown
---
title: Your Prompt Title
author: Your Name
author_link: https://twitter.com/yourhandle  # optional
model: HappyHorse
category: cinematic
video_url: https://...  # optional, link to generated video
date: 2026-04-09
---

## Description

A brief description of what this prompt generates.

## Prompt

Your full prompt text here...

## Preview

![Preview](url-to-image)
```

### Submit via Issue

Don't want to deal with PRs? Just [open an issue](https://github.com/Ericgood/happyhorse-prompt/issues/new) with your prompt and we'll add it for you!

### Guidelines

- ✅ Original prompts you created or have permission to share
- ✅ Include the model used (e.g., HappyHorse)
- ✅ Add a description of the expected output
- ✅ Video/image previews are highly encouraged
- ❌ No NSFW content
- ❌ No copyrighted character reproductions
